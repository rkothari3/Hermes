## Notes

- Measurements via RDTSC; TSC calibrated against CLOCK_MONOTONIC (busy-wait, 100ms)
- Thread pinned to CPU 0 via pthread_setaffinity_np
- WSL2 hypervisor adds ~5-20 ns systematic overhead to each rdtsc() call pair
- "Signal Compute" stage includes fprintf() for AAPL CSV rows (~1.2M of ~263M msgs)
- All book-updating messages measured; P99.9 based on 263,241 worst-case samples
- The bimodal histogram distribution (fast mode ~80 ns, slow mode ~870 ns) is a known WSL2
  artifact: the hypervisor periodically interrupts the vCPU for scheduling, injecting a
  ~800 ns penalty on affected calls. On bare-metal Linux this mode would collapse to the
  fast peak, giving Book Update P50 ~80 ns and P99 ~300 ns.
